## 중간고사 공부 내용
 저코드 / 노코드 지향(Ai agent)의 장점 : 개발 속도 향상, 초보자 접근성 높임

Java 기반 백엔드 프레임워크 특징 : 빠른 시작 시간과 낮은 메모리 사용 / 컨테이너 환경에서 실행 최적화 / GraalVm 네이티브 컴파일을 지원

컨테이너/Kubernetes, 서버리스 지원
장점 : Spring Boot 대비 10배 빠른 부팅, AI 통합 쉬움
단점 : 빌드 시간이 매우 길어서 프로젝트 이전이 어려움

Quarkus 기본 포트 번호 : 8080

!DOCTYPE : 문서 형식 정의
html : 모든 HTML 요소의 최상위 컨테이너
head : 브라우저 화면에 보이지 않는 메타데이터(설정 값), 문서 제목, 스타일 시트 등을 담음
meta : 페이지의 주요 전역 정보 / 문자 인코딩, 화면 배율, 검색 엔진 최적화 정보
title : 브라우저 상단 탭에 표시되는 제목
body : 사용자에게 실제로 보이는 모든 콘텐츠(텍스트, 이미지, 버튼 등)
div : 요소 그룹화, 레이아웃을 나누는 박스 모델(구분 짓는 역할?)
h1 : 문서 내에서 가장 중요한 제목(숫자가 커질수록 중요도는 낮아짐)
p : 텍스트 문단 구분
code : 프로그래밍 코드나 경로명 나타날 때 사용, 보통 고정폭 글꼴로 렌더링
link : 외부 파일 연결 / head 안에 작성, href 속성으로 경로 지정
script : 자바스크립트 연결
nav(네비게이션): 상단 메뉴나 링크 모음 구역임을 나타내는 의미론적 태그 // 웹사이트 길잡이 역할
container : 콘텐츠를 화면 중앙에 배치하고 좌우 여백을 자동으로 조절함 // Bootstrap의 가장 기본 박스
row / col (그리드 클래스) : 요소를 가로로 한 줄 세우고 칸의 너비를 나눔 // 한 줄을 12칸으로 나누어 배치
navbar-dark : 네비게이션 어두운 테마 // bg-dark와 함께 사용
img(클래스 추가) : 클래스를 통해 틀에 맞는 이미지 크기를 가짐
transition : 전환 효과 // 속성이 변할 때 부드럽게 바뀌는 시간 설정

html 화면을 채우는 요소는 문자 or 이미지
div 영역은 여러 개의 영역이 중첩되어 표현됨
개발자 모드 = f12

g-4(gutter 클래스) : 컬럼 사이 여백 조절 / 카드, 컬럼 사이 간격이 넓어지거나 좁아짐 / 카드가 너무 붙거나 너무 멀어 보임 조절 효과

row-cols-*(row columns 클래스) : 한 행에 들어갈 컬럼 개수 조절 / 화면 크기에 따라 1열 -> 5열로 자동 변경 / 모바일에선 세로, 데스크톱에선 가로 채움 효과

id : #으로 선택 | 한 문서에 단 하나의 요소에만 적용 | 특정 요소에 이름을 붙이는 데 사용

class : .으로 선택 | 동일한 값을 갖는 요소가 많음 | 스타일의 분류

상대 경로 : ./(생략 가능) ../
절대 경로 : http / /

html -> structure / css -> style / javascript -> behavior

외부 스크립트(External JS) : script...로 js 파일 분리 / 장점 : 유지보수 용이, 코드 재사용 가능 - 캐시 활용 가능 / 단점 : 초기 설정 필요, 파일 관리 필요 / 가장 기본적이고 권장하는 방식

상수로 지정해서 사용할 때는 const 사용 / 할당할 땐 let / 재선언할 땐 var

form : 데이터를 전송하고 페이지 새로고침
Call Stack : 현재 실행 중인 함수를 기록
Web APIs : 브라우저 제공 가능
Task Queue : 실행 대기 중인 콜백 함수 저장소
Event Loop : Stack과 Queue 사이의 교통 정리

div(division) : 무의미 태그 | 레이아웃 구성, css 스타일링 그룹화 목적
section : 의미론적 태그 | 문서의 주제별 구획 나눔 목적
